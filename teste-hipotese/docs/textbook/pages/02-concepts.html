Teste de Hipótese em ML

Apostila Interativa

Conceitos Fundamentais

H0, H1, p-valor, alfa, erros e poder estatístico — a base teórica completa, com visualizações interativas

O teste de hipótese é estruturado em torno de duas afirmações mutuamente exclusivas. Todo experimento começa com uma hipótese nula (H0) que representa o status quo — a ideia de que nada de especial está acontecendo — e uma hipótese alternativa (H1) que expressa o que gostaríamos de demonstrar.

Hipótese Nula (H0)

Não há diferença real entre os modelos. Qualquer diferença observada é fruto do acaso.

$H_0: \mu_A = \mu_B$
Hipótese Alternativa (H1)

Existe uma diferença real. O desempenho de um modelo é genuinamente diferente do outro.

$H_1: \mu_A \neq \mu_B$

A analogia do tribunal

Presunção de inocência

Em um julgamento, o réu é considerado inocente até prova em contrário. Da mesma forma, em estatística, assumimos que H0 é verdadeira até que as evidências demonstrem o contrário com confiança suficiente.

O veredicto

O júri pode condenar (rejeitar H0) ou absolver (não rejeitar H0). Note: absolver não é o mesmo que “provar inocência” — é apenas dizer que não há evidência suficiente para condenar.

Atenção

“Não rejeitar H0” NÃO é o mesmo que “aceitar H0”. Assim como absolver um réu não prova sua inocência, não rejeitar H0 significa apenas que não temos evidência suficiente contra ela.

2.1 Por que precisamos de múltiplas medições?

Imagine que o Random Forest obteve AUC = 0.832 e o MLP obteve AUC = 0.845 em um único conjunto de teste. O MLP é melhor?

Impossível dizer com apenas um número. Com uma única medição por modelo, não temos informação sobre variabilidade. Não sabemos se aquela diferença de 0.013 é estável ou se, ao repetir o experimento, ela poderia se inverter.

Com K-Fold Cross-Validation, treinamos e avaliamos cada modelo K vezes, obtendo K medições pareadas. Agora sim temos informação suficiente para calcular:

1
Média das diferenças ($\bar{d}$): quanto, em média, um modelo supera o outro ao longo dos folds.
2
Desvio padrão das diferenças ($s_d$): quão estável é essa vantagem — se varia muito entre folds, a diferença pode ser ruído.
3
Estatística de teste: combina $\bar{d}$ e $s_d$ para quantificar se a diferença é significativa dado o tamanho da amostra.
Pense assim

Com uma única medição, você tem um ponto no espaço. Sem referência, é impossível saber se ele está “longe” ou “perto” de algo. Com múltiplas medições, você vê a dispersão e pode inferir se um padrão é real ou apenas acaso.

2.2 O Teorema Central do Limite

O Teorema Central do Limite (TCL) afirma que, independentemente da forma da distribuição original dos dados, a distribuição das médias amostrais converge para uma distribuição normal conforme o número de amostras aumenta.

$$\sigma_{\bar{x}} = \frac{\sigma}{\sqrt{n}}$$

Onde σ é o desvio padrão da população e n é o tamanho de cada amostra. Quanto maior n, menor a variabilidade das médias amostrais.

Demonstração do Teorema Central do Limite
A população tem distribuição uniforme (não-normal). Observe como as médias amostrais formam uma curva normal.
Amostras maiores produzem médias mais concentradas ao redor da média real.

Quanto mais amostras você coleta, mais o histograma se aproxima de uma curva normal — mesmo que a população original seja uniforme. Esta é a essência do Teorema Central do Limite.

Por que isso importa?

O TCL é a razão pela qual podemos usar testes como o t-test mesmo quando os dados originais não seguem uma distribuição normal. As médias dos folds do K-Fold CV aproximam-se de uma normal, justificando o uso de testes paramétricos.

Regra prática

Para amostras de tamanho n ≥ 30, a aproximação normal é geralmente adequada. Em K-Fold CV com k = 10, temos apenas 10 medições — por isso usamos a distribuição t (que tem caudas mais largas) em vez da normal.

2.3 O P-valor

O p-valor é provavelmente o conceito mais importante — e mais mal-interpretado — da estatística. Vamos desdobrá-lo com cuidado.

Definição formal

O p-valor é a probabilidade de observarmos um resultado tão ou mais extremo que o obtido, assumindo que H0 é verdadeira — ou seja, que não há diferença real entre os modelos.

Desdobrando a definição

  1. Assumimos que H0 é verdadeira: começamos supondo que os modelos são iguais.
  2. Calculamos a probabilidade: dado que são iguais, qual a chance de vermos uma diferença tão grande?
  3. Se essa probabilidade for muito baixa: concluímos que os resultados seriam improváveis sob H0, e portanto rejeitamos H0.

Exemplo concreto

Suponha que a diferença média de AUC entre MLP e Random Forest em 10 folds foi 0.015. O p-valor = 0.032 nos diz: se os modelos fossem realmente iguais, haveria apenas 3,2% de chance de observarmos uma diferença de 0.015 ou maior. Como 3,2% é menor que nosso limiar de 5%, rejeitamos H0.

O que o p-valor NÃO é

NÃO é a probabilidade de H0 ser verdadeira

P = 0.03 não significa “3% de chance de H0 ser verdadeira”. O p-valor é calculado assumindo que H0 é verdadeira — ele não mede a probabilidade de H0 em si.

NÃO indica tamanho do efeito

P = 0.001 não significa que a diferença é grande. Um p-valor pequeno com uma diferença minúscula de AUC (ex: 0.001) pode não ter relevância prática alguma.

NÃO é binário

P = 0.049 e P = 0.051 são praticamente a mesma evidência. Tratar 0.05 como limiar rígido é uma simplificação — o p-valor é um espectro contínuo de evidência.

2.4 O Teste t Pareado

Agora que entendemos o p-valor conceitualmente, como calculamos ele na prática? O teste t pareado é o método mais comum para comparar dois modelos usando K-Fold CV.

A intuição

O teste t responde uma pergunta simples: a média das diferenças observadas é grande o suficiente, considerando a variabilidade, para ser improvável sob H0?

Passo a passo do cálculo

1
Calcule as diferenças pareadas: para cada fold $i$, compute $d_i = \text{score}_A(i) - \text{score}_B(i)$
2
Calcule a média das diferenças: $\bar{d} = \frac{1}{k} \sum d_i$
3
Calcule o desvio padrão: $s_d = \sqrt{\frac{1}{k-1} \sum (d_i - \bar{d})^2}$
4
Calcule a estatística t: $t = \frac{\bar{d}}{s_d / \sqrt{k}}$
5
Graus de liberdade: $df = k - 1$
6
Consulte a distribuição t com df graus de liberdade para obter o p-valor.

Graus de liberdade (df)

Com $k$ medições e 1 parâmetro estimado (a média), restam $k - 1$ valores “livres”. Quanto menor $df$, mais largas são as caudas da distribuição $t$ — exigindo uma diferença maior para rejeitar H0. Com $df \to \infty$, a distribuição $t$ converge para a normal.

Exemplo numérico

Cálculo passo a passo

Diferenças de F1-score em 10 folds:
[0.012, -0.003, 0.018, 0.005, 0.021, -0.001, 0.015, 0.008, 0.011, 0.007]

  • $\bar{d} = 0.0093$
  • $s_d = 0.0082$
  • $t = 0.0093 / (0.0082 / \sqrt{10}) = $ 3.59
  • $df = 9$, $p \approx$ 0.006

Com p = 0.006 < 0.05, rejeitamos H0: há evidência de diferença significativa entre os modelos.

2.5 Teste Unilateral vs. Bilateral

Até agora, usamos testes bilaterais (two-tailed) — testamos se há diferença em qualquer direção. Porém, existem situações onde faz sentido testar apenas uma direção.

Bilateral (two-tailed)
$H_1: \mu_A \neq \mu_B$

A região de rejeição está dividida nas duas caudas. Mais conservador, mais comum em ML comparativo.

Unilateral (one-tailed)
$H_1: \mu_B > \mu_A$

Toda a região de rejeição está em uma cauda. Mais poder para detectar diferenças na direção esperada.

Recomendação

Na maioria dos casos em ML, use o teste bilateral. Use unilateral apenas quando há uma hipótese clara a priori de que um modelo específico é superior — e registre essa decisão antes de ver os resultados.

2.6 O Nível de Significância (α)

O nível de significância α é o limiar que usamos para decidir se o p-valor é “pequeno o suficiente” para rejeitar H0.

$p < \alpha \rightarrow \text{rejeitar } H_0$   |   $p \ge \alpha \rightarrow \text{não rejeitar } H_0$

O valor α = 0.05 é uma convenção amplamente usada, mas não é uma lei da natureza. Significa que aceitamos um risco de 5% de cometer um erro Tipo I (rejeitar H0 quando ela é verdadeira).

Áreas mais rigorosas usam α = 0.01. Estudos exploratórios podem usar α = 0.10. A escolha deve ser feita ANTES de analisar os dados.

Cuidado com o p-hacking

Nunca ajuste α depois de ver o p-valor. Definir α = 0.06 porque seu p-valor deu 0.058 é manipulação estatística (p-hacking). O nível de significância deve ser pré-registrado.

2.7 Erros Tipo I e Tipo II

Qualquer decisão baseada em dados pode estar errada. Em testes de hipótese, existem exatamente dois tipos de erro possíveis.

Erro Tipo I — Falso Positivo

Rejeitar H0 quando ela é verdadeira. Afirmar que existe diferença entre os modelos quando, na realidade, eles são equivalentes.

$P(\text{Erro Tipo I}) = \alpha$
Erro Tipo II — Falso Negativo

Não rejeitar H0 quando ela é falsa. Concluir que os modelos são equivalentes quando, na verdade, um é melhor que o outro.

$P(\text{Erro Tipo II}) = \beta$

Analogias do dia a dia

Alarme de incêndio

Tipo I: alarme dispara sem fogo (falso alarme — incômodo mas seguro).
Tipo II: alarme NÃO dispara com fogo (perigo real — consequências graves).

Teste médico

Tipo I: diagnosticar doença em paciente saudável (tratamento desnecessário, ansiedade).
Tipo II: não diagnosticar doença real (sem tratamento, progressão da doença).

Em medicina, o Tipo II costuma ser mais grave. Em ML, depende do contexto: em um sistema de fraude, um falso negativo (fraude não detectada) pode ser pior que um falso positivo (transação legítima bloqueada).

2.8 Poder Estatístico

O poder estatístico é a probabilidade de rejeitar H0 quando ela é realmente falsa — ou seja, a capacidade do teste de detectar uma diferença real quando ela existe.

$$\text{Poder} = 1 - \beta = P(\text{rejeitar } H_0 \mid H_1 \text{ verdadeira})$$

Convenção: um teste deve ter poder ≥ 0.80 (80%). Um teste com poder de 0.60 significa que, mesmo que os modelos SEJAM diferentes, você tem 40% de chance de não perceber.

Os quatro fatores que afetam o poder

Tamanho do efeito (d)

Diferenças maiores são mais fáceis de detectar. Se um modelo é muito melhor, o teste precisa de menos evidência.

Variabilidade (σ)

Menos ruído nos dados facilita a detecção. Se os scores variam pouco entre folds, pequenas diferenças ficam evidentes.

Tamanho da amostra (k)

Mais medições (mais folds) fornecem mais informação. Por isso 10-fold CV é preferível a 5-fold quando possível.

Nível de significância (α)

Valores maiores de α (ex: 0.10 vs 0.05) aumentam o poder, mas também aumentam o risco de erro Tipo I. É um tradeoff.

2.9 Análise de Poder (Power Analysis)

A análise de poder permite planejar seu experimento ANTES de executá-lo, determinando quantas medições são necessárias para ter uma chance razoável de detectar uma diferença real.

Os quatro parâmetros interligados

São quatro grandezas interdependentes — dados três, calculamos o quarto:

Tamanho do efeito (d)

$d = \frac{\mu_1 - \mu_2}{\sigma}$

Convenções de Cohen: pequeno (0.2), médio (0.5), grande (0.8)

Nível $\alpha$

$\alpha = 0.05$

Geralmente 0.05. Controla a taxa de falsos positivos.

Poder desejado

$1 - \beta \ge 0.80$

Geralmente 0.80 (mínimo recomendado) ou 0.90 (ideal).

Tamanho da amostra ($n$)

$n = ?$

O que geralmente queremos calcular. Quantos folds ou repetições precisamos?

Workflow prático

1
Defina o efeito mínimo relevante: qual a menor diferença de desempenho que justifica escolher um modelo sobre outro? Ex: se 0.5% de diferença em AUC não muda nada na prática, o efeito mínimo pode ser d = 0.3.
2
Fixe α = 0.05 (ou o valor adequado ao seu contexto).
3
Escolha o poder desejado: 0.80 é o mínimo recomendado, 0.90 é ideal.
4
Calcule n: use a fórmula ou ferramentas como statsmodels em Python.

Implementação em Python

Python
from statsmodels.stats.power import TTestIndPower

# Criar o analisador de poder
analysis = TTestIndPower()

# Calcular o tamanho da amostra necessário
n = analysis.solve_power(
    effect_size=0.5,   # Efeito médio (Cohen)
    alpha=0.05,        # Nível de significância
    power=0.80,        # Poder desejado
    ratio=1.0          # Razão entre grupos
)
print(f"Amostras necessárias por grupo: {n:.0f}")  # ~64

# Calcular o poder dado n fixo
power = analysis.solve_power(
    effect_size=0.5,
    alpha=0.05,
    nobs1=10,          # Ex: 10-fold CV
    ratio=1.0
)
print(f"Poder com k=10 folds: {power:.3f}")  # ~0.29
Limitação do K-Fold CV

Com k = 10 folds, o poder para detectar efeitos médios (d = 0.5) é de apenas ~29%. Para detectar diferenças pequenas entre modelos, considere usar 5×2 CV (5 repetições de 2-fold), repeated k-fold (múltiplas repetições do 10-fold), ou bootstrap pareado.

Dica prática

Antes de comparar modelos, faça uma power analysis. Se o poder for muito baixo com o design experimental escolhido, aumente k, use repetições, ou aceite que só poderá detectar diferenças grandes.

2.10 Resumo do capítulo

Conceitos fundamentais para levar
  • H0 assume que não há diferença; H1 propõe que a diferença é real.
  • Precisamos de múltiplas medições (K-Fold CV) para ter variabilidade e poder testar hipóteses.
  • O Teorema Central do Limite justifica o uso de testes paramétricos com médias amostrais.
  • O p-valor mede a probabilidade dos dados sob H0 — NÃO a probabilidade de H0 ser verdadeira.
  • O teste t pareado transforma diferenças observadas em uma estatística comparável.
  • α controla o erro Tipo I; β controla o erro Tipo II; há um tradeoff inevitável entre eles.
  • Poder = 1 − β é a capacidade de detectar diferenças reais — mire em ≥ 0.80.
  • Power analysis antes do experimento evita testes com poder insuficiente.
