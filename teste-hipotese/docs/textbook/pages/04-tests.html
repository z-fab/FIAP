Apostila Interativa

Testes Estatísticos para Comparação de Modelos

Escolhendo e aplicando o teste correto para provar que um modelo é superior a outro

Você treinou dois modelos, calculou as métricas e um parece melhor. Mas essa diferença é real ou apenas ruído da amostragem? Neste capítulo, vamos percorrer os principais testes estatísticos usados para comparar modelos de machine learning, desde o clássico paired t-test até abordagens robustas como o 5x2 CV. Cada teste tem premissas, vantagens e limitações — escolher o teste errado pode invalidar completamente sua conclusão.

4.1 Paired t-test (Teste t Pareado)

O paired t-test é o teste mais utilizado para comparar dois modelos em ML. A ideia é simples e elegante: executamos K-Fold Cross-Validation e, para cada fold, temos o F1-Score (ou outra métrica) de ambos os modelos. Como os mesmos dados de teste são usados para avaliar ambos em cada fold, as observações são pareadas.

Procedimento do Paired t-test
  1. Executar K-Fold CV (tipicamente K=10) com os dois modelos nos mesmos folds
  2. Para cada fold i, calcular a diferença: d_i = metrica_A_i - metrica_B_i
  3. Calcular a média das diferenças: d_bar = media(d_1, ..., d_K)
  4. Calcular o desvio-padrão das diferenças: s_d
  5. Calcular a estatística t: t = d_bar / (s_d / sqrt(K))
  6. Comparar com a distribuição t de Student com K-1 graus de liberdade
$$t = \frac{\bar{d}}{s_d / \sqrt{K}} \quad \text{onde} \quad \bar{d} = \frac{1}{K}\sum_{i=1}^{K} d_i \quad \text{e} \quad s_d = \sqrt{\frac{1}{K-1}\sum_{i=1}^{K}(d_i - \bar{d})^2}$$

As hipóteses são:

  • H0: A média das diferenças é zero (os modelos são equivalentes)
  • H1: A média das diferenças é diferente de zero (um modelo é melhor)
Premissa de Normalidade

O paired t-test assume que as diferenças d_i seguem uma distribuição normal. Com K=10 folds, temos apenas 10 observações, o que torna difícil verificar normalidade. Na prática, o teste é razoavelmente robusto a desvios moderados de normalidade (pelo Teorema Central do Limite), mas em amostras muito pequenas (K=5) ou com diferenças muito assimétricas, considere usar o teste de Wilcoxon (seção 4.2).

Problema: Dependência entre Folds

O paired t-test assume que as diferenças d_i são independentes. No K-Fold CV, os conjuntos de treino compartilham a maioria dos dados entre si (no 10-Fold, cada par de folds compartilha ~80% dos dados de treino). Isso viola a premissa de independência e faz o teste subestimar a variância, resultando em mais falsos positivos do que o esperado. O 5x2 CV t-test (seção 4.5) foi criado para resolver exatamente este problema. Isso é conhecido como inflação de Erro Tipo I.

4.2 Teste de Wilcoxon Signed-Rank

O teste de Wilcoxon é a alternativa não-paramétrica ao paired t-test. Em vez de assumir normalidade das diferenças, ele trabalha com os ranks (postos) dos valores absolutos das diferenças.

Procedimento do Wilcoxon Signed-Rank
  1. Calcular as diferenças pareadas: d_i = metrica_A_i - metrica_B_i
  2. Descartar diferenças iguais a zero
  3. Ordenar os valores absolutos |d_i| e atribuir ranks (1, 2, ..., n)
  4. Somar os ranks das diferenças positivas: W+ = soma dos ranks onde d_i > 0
  5. Somar os ranks das diferenças negativas: W- = soma dos ranks onde d_i < 0
  6. A estatística do teste é W = min(W+, W-)
  7. Comparar W com a distribuição tabelada (ou usar aproximação normal para n > 20)

Quando usar Wilcoxon em vez do t-test:

  • As diferenças não são aproximadamente normais (distribuição assimétrica ou com outliers)
  • Poucos folds (K=5), onde normalidade é difícil de justificar
  • Você quer um teste mais conservador (menos falsos positivos)
  • As métricas são ordinais ou baseadas em ranks (ex: posição no ranking de competição)
Trade-off: Robustez vs Poder

O Wilcoxon não assume normalidade, mas paga um preço: tem menor poder estatístico que o t-test quando os dados realmente são normais. Isso significa que, se a diferença entre os modelos for pequena mas real, o t-test tem mais chance de detectá-la. Se você tem K >= 10 folds e as diferenças parecem razoavelmente simétricas, o t-test provavelmente é a melhor escolha.

4.3 Teste de McNemar

Os testes anteriores (t-test, Wilcoxon) comparam métricas agregadas por fold. O teste de McNemar é fundamentalmente diferente: ele compara as predições amostra por amostra em um único conjunto de teste.

Tabela de Contingência do McNemar
Modelo B: CorretoModelo B: Errado
Modelo A: Correto$e_{00}$ (ambos acertam)$e_{01}$ (só A acerta)
Modelo A: Errado$e_{10}$ (só B acerta)$e_{11}$ (ambos erram)
$$\chi^2 = \frac{(|e_{01} - e_{10}| - 1)^2}{e_{01} + e_{10}} \quad \text{com 1 grau de liberdade}$$

O teste ignora os casos em que ambos acertam ou ambos erram ($e_{00}$ e $e_{11}$) — eles não trazem informação sobre a diferença entre os modelos. Foca nos casos discordantes: $e_{01}$ (só A acerta) e $e_{10}$ (só B acerta). Se a proporção de discordâncias é significativamente diferente de 50/50, concluímos que os modelos são diferentes.

Vantagens do McNemar:

  • Não requer cross-validation: Funciona com um único split treino/teste
  • Granularidade por amostra: Usa cada predição individualmente, maximizando informação
  • Sem premissa de normalidade: Baseado na distribuição qui-quadrado
  • Simples de implementar: Precisa apenas da tabela 2x2
Limitação do McNemar

O McNemar depende de um único conjunto de teste. Se esse conjunto não for representativo da população (por azar da amostragem), a conclusão pode ser enganosa. Os testes baseados em K-Fold são mais robustos nesse aspecto, pois avaliam em K conjuntos de teste diferentes. Use McNemar quando cross-validation é impraticável (modelos muito caros de treinar, como LLMs).

4.4 Teste de Friedman + Post-hoc de Nemenyi

Quando você tem 3 ou mais modelos para comparar, fazer testes pareados entre todos os pares (seção 4.6) infla a taxa de falsos positivos. O teste de Friedman é a alternativa correta: é o equivalente não-paramétrico da ANOVA de medidas repetidas.

Procedimento
  1. Executar K-Fold CV com todos os M modelos nos mesmos folds
  2. Para cada fold, rankear os modelos de 1 (melhor) a M (pior)
  3. Calcular o rank médio de cada modelo: R_j = (1/K) ∑ r_ij
  4. Calcular a estatística de Friedman:
    $$\chi^2_F = \frac{12K}{M(M+1)}\left[\sum_{j=1}^{M} R_j^2 - \frac{M(M+1)^2}{4}\right]$$
  5. Se Friedman rejeitar H0 (os modelos não são todos iguais), aplicar Nemenyi post-hoc para identificar quais pares diferem

O teste de Nemenyi calcula a diferença crítica (CD): se a diferença entre os ranks médios de dois modelos excede CD, eles são significativamente diferentes. Os resultados são frequentemente visualizados com um diagrama CD, onde modelos conectados por uma barra horizontal não diferem significativamente.

Exemplo Prático

Você compara Random Forest, XGBoost, LightGBM e uma MLP em 10 folds. Os ranks médios são: XGBoost (1.4), LightGBM (1.8), RF (2.8), MLP (4.0). O teste de Friedman rejeita H0 (p = 0.003). O Nemenyi com CD = 1.5 indica que: XGBoost e LightGBM não diferem (diferença = 0.4 < 1.5), mas ambos são melhores que a MLP (diferenças > 1.5). O RF fica em zona cinzenta.

4.5 Teste t de 5x2 CV (Dietterich)

O teste 5x2 CV, proposto por Dietterich (1998), foi especificamente projetado para resolver o problema de dependência do K-Fold mencionado na seção 4.1. Em vez de usar 10-Fold CV, ele executa 5 repetições de 2-Fold CV.

Procedimento do 5x2 CV t-test
  1. Repetir 5 vezes (i = 1, ..., 5):
    • Dividir os dados aleatoriamente em duas metades (S1 e S2)
    • Treinar modelo A em S1, testar em S2 → p_A1. Treinar em S2, testar em S1 → p_A2
    • Fazer o mesmo para modelo B → p_B1, p_B2
    • Calcular diferenças: d_i1 = p_A1 - p_B1 e d_i2 = p_A2 - p_B2
  2. Calcular a variância de cada repetição: s_i^2 = (d_i1 - d_bar_i)^2 + (d_i2 - d_bar_i)^2
  3. Estatística do teste: t = d_11 / sqrt((1/5) * sum(s_i^2))
  4. Distribuição: t de Student com 5 graus de liberdade

Por que 5x2 e não 10-Fold?

  • 2-Fold: Cada fold usa 50% dos dados para treino, sem sobreposição entre folds. As duas estimativas em cada repetição são genuinamente independentes.
  • 5 repetições: Diferentes embaralhamentos fornecem variabilidade. Mais repetições não ajudam muito (rendimentos decrescentes).
  • Menos viés: O treino com 50% dos dados produz estimativas com mais viés (modelos piores), mas a variância da estimativa do teste é mais honesta.
Recomendação

Se você pode arcar com o custo computacional (treinar cada modelo 10 vezes), o 5x2 CV t-test é geralmente a melhor opção para comparar dois modelos. Ele tem a taxa de Tipo I (falso positivo) mais próxima do nível nominal de 5% entre todos os testes discutidos.

4.6 O Problema das Comparações Múltiplas

Imagine que você treinou 4 modelos e quer saber se algum par difere significativamente. São C(4,2) = 6 pares para testar. Mesmo que todos os modelos sejam idênticos, cada teste tem 5% de chance de gerar um falso positivo. A probabilidade de pelo menos um falso positivo é:

$$P(\text{pelo menos 1 FP}) = 1 - (1 - 0.05)^6 = 1 - 0.95^6 = 0.265 = \mathbf{26{,}5\%}$$

Ou seja, em mais de 1 em cada 4 experimentos, você publicará uma "descoberta" que é puro acaso. Com 10 modelos (45 pares), essa probabilidade sobe para 90%. Isso é o problema das comparações múltiplas, e ignorá-lo é um dos erros mais comuns em papers de ML.

O Erro que Reprova no Peer Review

Relatórios e papers que comparam múltiplos modelos sem correção para comparações múltiplas são frequentemente rejeitados em conferências como NeurIPS, ICML e KDD. Revisores experientes checam isso rotineiramente. Na PosTech, aplicar a correção correta demonstra maturidade estatística.

Correções para Comparações Múltiplas

Bonferroni

A correção mais simples e conservadora. Divide o nível de significância pelo número de comparações: α_adj = α / m. Para 6 comparações com α = 0.05: α_adj = 0.0083. Muito conservador — pode não detectar diferenças reais.

Holm-Bonferroni

Versão sequencial do Bonferroni. Ordena os p-values e aplica thresholds progressivamente menos rigorosos. Uniformemente mais poderoso que Bonferroni. Recomendado como padrão.

Nemenyi (post-hoc)

Específico para o contexto do teste de Friedman. Baseado na distribuição Studentized Range. Controla o family-wise error rate para comparações de ranks.

4.7 Tamanho do Efeito (Cohen's d)

O p-valor responde "existe diferença?", mas não diz "quão grande é essa diferença?". O tamanho do efeito (effect size) complementa o teste de hipótese medindo a magnitude prática da diferença.

Para comparações pareadas (como K-Fold CV), usamos o Cohen's d:

$$d = \frac{\bar{d}}{SD(d)}$$

Onde d̄ é a média das diferenças pareadas e SD(d) é o desvio padrão dessas diferenças.

Pequeno (d ≈ 0.2)

Diferença real mas difícil de perceber na prática. Pode não justificar trocar de modelo.

Médio (d ≈ 0.5)

Diferença perceptível e provavelmente relevante. Vale considerar a troca.

Grande (d ≈ 0.8)

Diferença substancial e claramente relevante. Forte evidência para preferir um modelo.

⚠️ Significância Estatística ≠ Significância Prática

Cenário 1: p = 0.001, d = 0.05 → Estatisticamente significativo, mas a diferença é negligível. Não vale trocar o modelo.

Cenário 2: p = 0.03, d = 0.8 → Significativo E com efeito grande. Evidência forte para preferir um modelo.

Sempre reporte o Cohen's d junto com o p-valor!
