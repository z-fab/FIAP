Teste de Hipótese em ML

Apostila Interativa

O Fluxo Completo

Um protocolo de 6 passos para comparar modelos com rigor estatístico usando K-Fold Cross-Validation

Nos capítulos anteriores, estabelecemos por que o teste de hipótese é necessário e quais conceitos o fundamentam. Agora é hora de colocar tudo junto em um fluxo de trabalho completo e prático que você pode aplicar em qualquer projeto de machine learning.

O protocolo que vamos detalhar a seguir é baseado na abordagem mais aceita na literatura: comparação pareada usando Stratified K-Fold Cross-Validation. A ideia central é que ambos os modelos são avaliados nos mesmos folds de dados, o que nos permite calcular diferenças pareadas e aplicar testes estatísticos com mais poder.

Por que pareado?

Quando dois modelos são avaliados nos mesmos folds, as flutuações causadas pela "dificuldade" de cada fold afetam ambos igualmente. Ao calcular a diferença entre os modelos em cada fold, eliminamos essa fonte de variação compartilhada (diferenças pareadas). Isso é análogo a um estudo clínico onde cada paciente recebe ambos os tratamentos (crossover), em vez de grupos separados.

3.1 Os 6 Passos

1

Definir o Stratified K-Fold

O primeiro passo é definir a estratégia de validação cruzada. Usamos Stratified K-Fold porque:

  • Estratificação garante que cada fold mantém a proporção de classes do dataset original. Isso é especialmente importante em problemas desbalanceados (ex: 95% não-churn, 5% churn).
  • K = 10 é o valor mais comum na prática. Oferece um bom equilíbrio entre bias (baixo com K alto) e variância (baixa com K baixo). Dietterich (1998) e Demšar (2006) recomendam K = 10.
  • Mesmos folds para todos os modelos: este é o ponto crucial. Usamos a mesma seed e a mesma partição para todos os modelos, garantindo que as comparações sejam pareadas.
Python
from sklearn.model_selection import StratifiedKFold

skf = StratifiedKFold(n_splits=10, shuffle=True, random_state=42)

# Os mesmos indices para TODOS os modelos:
folds = list(skf.split(X, y))
2

Treinar todos os modelos nos mesmos splits

Para cada um dos K folds, treinamos todos os modelos usando o mesmo conjunto de treino e avaliamos no mesmo conjunto de teste. Isso cria uma correspondência 1-a-1 entre as avaliações.

Cuidados importantes:

  • Pré-processamento dentro do fold: qualquer transformação (scaling, encoding, feature selection) deve ser feita dentro de cada fold, usando apenas os dados de treino. Vazamento de dados aqui invalida todo o experimento.
  • Hiperparâmetros: se os modelos já foram tunados, use os hiperparâmetros finais. Se quer comparar pipelines completas (incluindo tuning), faça o tuning dentro de cada fold (nested CV).
  • Seeds internas: para modelos estocásticos (redes neurais, random forests), fixe a seed interna para reduzir variância. Mas lembre-se que isso só fixa uma fonte de variação, não elimina a variabilidade dos folds.
Python
from sklearn.metrics import roc_auc_score

scores_rf = []
scores_mlp = []

for train_idx, test_idx in folds:
    X_train, X_test = X[train_idx], X[test_idx]
    y_train, y_test = y[train_idx], y[test_idx]

    # Treinar e avaliar RF
    rf.fit(X_train, y_train)
    scores_rf.append(roc_auc_score(y_test, rf.predict_proba(X_test)[:,1]))

    # Treinar e avaliar MLP
    mlp.fit(X_train, y_train)
    scores_mlp.append(roc_auc_score(y_test, mlp.predict_proba(X_test)[:,1]))

4

Calcular diferenças pareadas

Para cada fold, calculamos a diferença: $d_i = \text{score}_{MLP,i} - \text{score}_{RF,i}$. Essas diferenças são a matéria-prima do teste estatístico.

$$\bar{d} = \frac{1}{K}\sum_{i=1}^{K} d_i \qquad s_d = \sqrt{\frac{1}{K-1}\sum_{i=1}^{K}(d_i - \bar{d})^2}$$

Se a média das diferenças ($\bar{d}$) é grande em relação a $s_d$, temos evidência de diferença real. Se $\bar{d}$ é pequeno e $s_d$ é grande, a diferença observada é provavelmente ruído.

Intuição

Pense nas diferenças $d_i$ como "votos". Se todos os folds apontam na mesma direção (todos positivos ou todos negativos), a evidência é forte. Se metade é positiva e metade é negativa, não há consenso — a diferença pode ser acaso.

5

Aplicar o teste estatístico

O teste mais comum para comparação pareada de dois modelos é o teste t pareado:

$$t = \frac{\bar{d}}{s_d / \sqrt{K}} \quad \text{com } K-1 \text{ graus de liberdade}$$

Alternativamente, para evitar premissas de normalidade, pode-se usar o teste de Wilcoxon signed-rank (não-paramétrico).

Python
from scipy import stats
import numpy as np

diffs = np.array(scores_mlp) - np.array(scores_rf)

# Teste t pareado
t_stat, p_value = stats.ttest_rel(scores_mlp, scores_rf)

# Alternativa nao-parametrica
w_stat, p_wilcoxon = stats.wilcoxon(diffs)
Limitação do teste t pareado com K-Fold

O teste t pareado assume que as observações são independentes, mas os folds do K-Fold CV compartilham dados de treino, violando essa premissa. Isso pode levar a p-valores otimistas (rejeitar $H_0$ com mais frequência do que deveria). Alternativas mais robustas incluem o corrected resampled t-test de Nadeau & Bengio (2003) e o 5x2cv paired t-test de Dietterich (1998).

6

Interpretar e decidir

Com o p-valor em mãos, a decisão segue a lógica que já conhecemos:

Se p < α (ex: p = 0.02, α = 0.05)

Rejeitar $H_0$. Há evidência estatística de que os modelos têm desempenho diferente. Verifique a direção ($\bar{d} > 0$ ou $\bar{d} < 0$) para saber qual é melhor. Considere também o tamanho do efeito: a diferença é prática e relevante?

Se p ≥ α (ex: p = 0.38, α = 0.05)

Não rejeitar $H_0$. Não há evidência suficiente de diferença. Nessa situação, prefira o modelo mais simples, mais rápido ou mais interpretável — já que não há ganho estatisticamente comprovado em usar o mais complexo.

Significância estatística não é significância prática

Um p-valor de 0.001 com diferença média de 0.002 em AUC pode ser estatisticamente significativo, mas praticamente irrelevante. Sempre avalie se a magnitude da diferença justifica a troca de modelo, considerando custos computacionais, complexidade de deploy, interpretabilidade, etc.

3.4 Checklist final

Antes de reportar os resultados de qualquer comparação de modelos, verifique:

Verificações essenciais
  • Os modelos foram avaliados nos mesmos folds (mesma seed, mesma partição)?
  • O pré-processamento foi feito dentro de cada fold, sem vazamento?
  • O α foi definido antes de rodar o teste?
  • O teste estatístico é apropriado para a situação (pareado, quantidade de modelos, premissas)?
  • Além do p-valor, você reportou o tamanho do efeito (magnitude da diferença)?
  • Se comparou mais de 2 modelos, aplicou correção para múltiplas comparações?
  • Os resultados são reprodutíveis (seeds documentadas, código disponível)?
Para 3+ Modelos

Este fluxo de 6 passos é para comparação de 2 modelos. Para 3 ou mais modelos simultâneos, use o teste de Friedman + Nemenyi post-hoc (Capítulo 4, Seção 4.4). O Notebook 02 implementa este cenário com 9 configurações.

3.5 Resumo do capítulo

O fluxo em uma frase

Divida os dados → Treine nos mesmos folds → Colete scores por fold → Calcule diferenças pareadas → Aplique o teste → Interprete com cuidado.

Com este protocolo, você tem uma base sólida para comparar quaisquer dois modelos com rigor estatístico. Nos próximos capítulos, exploraremos testes específicos em detalhe (t-test pareado, Wilcoxon, McNemar, 5x2cv), bem como a comparação de múltiplos modelos simultaneamente (Friedman + Nemenyi).
